## Graphic Demonstration
## I. Install Hexo

### 1. Create a Folder

Create a folder anywhere and name it "hexo" (do not enter the double quotes)

### 2. Install Hexo with Git

Right-click in the "Hexo" folder and select "Open Git Bash here"

Enter the following codes in sequence in the opened command line:

```gui
npm install hexo-cli -g

hexo init blog

cd blog

npm install

hexo s
```

If the message "INFO Hexo is running at http://localhost:4000. Press Ctrl+C to stop." appears, it means the local operation is successful.

Enter [http://localhost:4000/](http://localhost:4000) in your browser to view your blog.

*"hexo s" is the abbreviation of "hexo server"

### II. Beautify the Blog

Open the Hexo Theme Center [Access here](https://hexo.io/themes/)

![1024](/img/1/1-1.avif)

Find the theme you like, click to enter and jump to Github, and check the theme code in the description bar.

Right-click in the hexo/blog directory and select "Open Git Bash here".

Paste the code into the pop-up command line and press Enter to run it.

After the code runs successfully, find "_config.yml" in the hexo/blog directory.

Open it with VS Code, find the "theme" field, and change the theme name after the colon to the name of the theme folder you installed in hexo\blog\themes.

After the above code runs successfully, right-click in the hexo/blog directory and select "Open Git Bash here".

Enter the following in sequence in the opened command line:

```txt
hexo g
hexo clean
hexo s
```

Then enter [http://localhost:4000/](http://localhost:4000/) in your browser to check if the theme of your blog has been modified successfully.

## III. Create a GitHub Repository

### 1. Log in to Github: [Official Website](https://github.com)

*Sometimes Github is inaccessible in China, you can use Watt Toolkit to accelerate access.

### 2. Create a New Repository

Change the repository name to "your username.github.io"

Then follow the default options all the way until the following page appears

![Github Create New Repository Page](/img/1/1-2.avif)

## IV. Use Git to Set an Identity and Obtain SSH Key

![Github Username](/img/1/1-3.avif)

Right-click in the hexo/blog directory and select "Open Git Bash here"

Enter the following codes in sequence:

```txt
git config --global user.name "Your Github Username"

git config --global user.email "The email you used to register on GitHub"

ssh-keygen -t rsa -C "The email you used to register on GitHub"

cat ~/.ssh/id_rsa.pub
```

If several lines of seemingly garbled key appear, it means success.

## V. Copy the Git SSH Key to GitHub

![Github SSH Key Copy Page](/img/1/1-4.avif)

## VI. Push Local Code to GitHub

Find "_config.yml" in the hexo/blog directory, locate the end of the file, and modify the last few lines of code to look like this:

```yml
deploy:
  type: 'git'
  repository: Your repository's SSH link
  branch: main
```

Right-click in the hexo/blog directory and select "Open Git Bash here"

Enter the following in sequence in the opened command line:

```txt
hexo g
hexo d
```

Wait for the code to finish running.

## VIII. Modify the Website in the Future

Every time you modify the website, right-click in the hexo/blog directory and select "Open Git Bash here"

Enter the following in sequence in the opened command line:

```txt
hexo g
hexo clean
hexo d
```

Now you have your own website, go and try it out!
