Cloudflare is a well-known free CDN service provider with nodes around the world (excluding mainland China), offering excellent international access speed. However, the user experience in China is unsatisfactory. Although upgrading to an enterprise plan can improve this issue, ordinary bloggers are unlikely to choose it. Today, I will share a practical **unconventional method** to help you improve Cloudflare's access speed in China.

### 2. Add the **Custom Domain** to **Cloud DNS Service**

Click to go to [Cloud DNS Service](https://console-intl.huaweicloud.com/dns), then click `Public Domain Names`.  

Click `Create Public Domain Name` in the upper right corner.  

Enter the **Pages custom domain** to be optimized (i.e., `cf.com`), then click `OK`.  

## Solution 1: Directly CNAME to an Optimized Domain

After my selection, only one CNAME domain meets my requirements: `cf.468123.xyz`

`cf.468123.xyz` supports IPv4 and IPv6, with good access speed both inside and outside China.

Log in to Huawei Cloud, click `Manage Resolution` for `cf.com`.  

After entering `Manage Resolution`, click `Add Record Set`.  

Add a new `CNAME Record` with the line type set to `Global Default` and the record value as the **Pages project-assigned domain** `w.pages.dev`.  

Add another `CNAME Record` with the line type set to `Geographic Resolution` > `Mainland China` and the record value as the **optimal domain** `cf.468123.xyz`.  

Finally, add the following Huawei Cloud NS addresses on Cloudflare:

```TXT
ns1.huaweicloud-dns.com.  
ns1.huaweicloud-dns.cn.  
ns1.huaweicloud-dns.net.  
ns1.huaweicloud-dns.org.
```

That's it.
